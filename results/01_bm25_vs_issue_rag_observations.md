# BM25 vs Issue-level RAG 실행 결과 비교

이 문서는 같은 RAG 파이프라인을 조금씩 바꿨을 때 실제 출력이 어떻게 달라지는지 보기 위한 수업용 실행 로그다.

수강생들이 먼저 아래 raw output을 보고 직접 판단하게 한 뒤, 마지막의 관찰 질문으로 토론하면 된다.

## 1. Raw Text BM25 RAG

구성:

```text
PDF page text -> chunking -> BM25 top-k -> LLM answer
```

실행 명령:


## 2. Issue BM25 RAG

구성:

```text
LLM-extracted issues -> issue-level BM25 top-k -> LLM answer
```

실행 명령:

```bash
sct-project-graphrag/.venv/bin/python sct-project-graphrag/scripts/10_bm25_issue_rag_answer.py \
  "사업자등록증과 통장 거래내역을 확인했으면 선의의 거래당사자로 인정될 수 있어?" \
  --top-k 5 \
  --raw-context none \
  --model gpt-5.4-mini
```

```text
정리하면, **사업자등록증과 통장 확인만으로는 부족하고**, 거래 상대방의 실체, 거래 조건의 정상성, 추가 확인 노력까지 종합해서 판단합니다[2][4][5].
```

## 3. Issue BM25 RAG + Same-case Raw Context

구성:

```text
Issue-level BM25 top-k
+ retrieved issue가 나온 같은 사건의 raw text chunk
-> LLM answer
```

실행 명령:

```bash
sct-project-graphrag/.venv/bin/python sct-project-graphrag/scripts/10_bm25_issue_rag_answer.py \
  "사업자등록증과 통장 거래내역을 확인했으면 선의의 거래당사자로 인정될 수 있어?" \
  --top-k 5 \
  --raw-context same-case \
  --max-raw-chunks-per-case 1 \
  --max-raw-context-chars 5000 \
  --model gpt-5.4-mini
```
